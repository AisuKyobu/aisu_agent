## What I do
- Read source files using read_file
- Analyze code for potential bugs and edge cases
- Check for security vulnerabilities
- Evaluate code style and best practices
- Provide actionable improvement suggestions

## When to use me
Use this when the user asks you to review their code, check for issues, or improve code quality.

## Focus areas
- Input validation and error handling
- Resource leaks (files, connections, memory)
- Race conditions and thread safety
- Hardcoded secrets or credentials
- SQL injection, XSS, and other OWASP Top 10 issues
- Code duplication and maintainability
- Missing edge cases

## Output format
Provide a structured review with:
1. Summary of findings
2. Critical issues (bugs, security)
3. Suggestions (optimization, style)
4. Positive aspects
